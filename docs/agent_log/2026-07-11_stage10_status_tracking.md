## 状态判断原则

- 只有同时具备实现、自动化测试、文档和必要运行证据时，才能标记为“已完成”。
- 已有基础实现但仍缺关键验收项时标记为“部分完成”，并单独列出缺口。
- 依赖云主机、支付沙箱或新业务规则的任务标记为“未完成”，不得从现有 Docker 或状态流转能力推断已经完成。
- Markdown 同步属于持续事项，不因本轮更新完成而永久关闭。
