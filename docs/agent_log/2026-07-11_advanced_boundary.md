# 2026-07-11 第十阶段进阶边界执行日志

## 目标

执行 `docs/project_evolution.md` 中“第十阶段：后续进阶边界”的可落地任务，并记录执行过程。

## 本轮完成内容

1. 修复多商品订单锁顺序风险：
   - `OrderService.CreateOrder` 在事务内复制订单明细并按 `product_id` 升序处理；
   - 下单请求商品顺序不同，也会以一致顺序读取商品和锁定库存行；
   - 新增服务层测试，验证倒序提交时订单明细和库存流水按 `product_id` 升序写入。

2. 新增后台操作审计：
   - 新增迁移 `migrations/00014_create_operation_logs.sql`；
   - 新增 `operation_logs` 模型、DAO、service、handler 和管理员查询接口；
   - 新增 `OperationAudit` 中间件，在管理员鉴权通过后记录后台操作；
   - 操作日志记录操作者、动作、路由、HTTP 状态、请求 ID、客户端 IP、User-Agent 和时间；
   - 不记录请求体，避免敏感信息落库；
   - 审计写入失败不影响原业务请求。

3. 新增 React 管理台操作日志页面：
   - 新增 `/operation-logs` 管理员页面；
   - 侧边栏仅管理员展示“操作日志”入口；
   - 页面展示管理员、动作、状态、路径、请求 ID、IP 和时间。

4. 同步 Markdown：
   - 更新接口清单、权限设计、表设计、业务规则、需求边界、测试计划、测试结果、README 和项目演进文档；
   - 新增 `docs/http/operation_logs.http` 方便手动接口验证。
